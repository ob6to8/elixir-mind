# elixir-mind

The operator's working name for this brain's prospective **owned runtime**: the resident BEAM system mapped as tiers 1–2 in [the BEAM/Jido evaluation](/meta/analysis/beam-deployment-and-jido-2-evaluation.md) — a supervised daemon owning scheduled automation and the embedding index, exposing the `brain.*` verbs over MCP, and eventually hosting [Jido](/beliefs/glossary/jido.md) agents as the mechanical plane of the [two-plane rule](/beliefs/glossary/two-plane-rule.md). The name marries [second mind](/beliefs/glossary/second-mind.md) (an AI layer that reasons over stored knowledge, not just stores it) with the Elixir substrate the brain's tooling is written in. It names a *direction with recorded adoption triggers*, not a built system: the term first appeared as a document rename that was applied and reverted (the 2026-07-13 taxonomy session), then stabilized as the project name in the 2026-07-14 AST/macros session, whose analysis derives its design constraints.
