# auto-intake

Its premise is that the [featuring](/beliefs/glossary/featuring.md) gates [`/research`](/.claude/skills/research/SKILL.md) applies are already the quality filter, so an item needs no second gate before [`/intake`](/.claude/skills/intake/SKILL.md) files it; the editorial pass then consists of pruning, relabeling, and merging. It is bounded to the known tree — items needing a new top-level directory are **deferred** for ratification, never filed autonomously — prefers update-in-place on a `relates to sb:` hint to keep [residual fragmentation](/beliefs/glossary/residual-fragmentation.md) small, and tags what it files `auto-intake` so the editorial queue is one query. The digest itself stays a non-bundle record; only its featured items acquire ids. The tradeoff it accepts is [cognitive debt](/beliefs/glossary/cognitive-debt.md): the corpus can grow faster than the operator internalizes it.
