# doctrine

The layer of guiding principles and accepted tenets in a governance system — the "why" that shapes judgment without prescribing specific action. Doctrine informs policy; a [policy](/glossary/policy-type.md) implements doctrine as a concrete, enforceable rule. This brain's controlled vocabulary has no `doctrine` type: the closest analogue is the preamble and compiled [operating contract](/glossary/operating-contract.md) read as a whole (its worldview — "the tree is the taxonomy", "distill, don't dump"), while each `type: policy` file is a single standing rule.
